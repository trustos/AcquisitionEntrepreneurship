# Acquisition Scorecard — pomocalm.com

**Verdict:** MARGINAL — 45.0/100 (completeness 54%)

The first target that's the right shape — a web app your team can own and scale, so the competence gate finally PASSES — but almost certainly abandoned pre-revenue code: zero public footprint, no metrics disclosed, red-ocean category. Only worth it if the seller proves a real, engaged userbase.

## Scores
- build_scale_fit: 5/5 (high) A productivity web app is dead-center in the team's wheelhouse — they can own, modernize, add AI, and scale it. Competence gate PASSES (first of the three candidates).
- pmf_retention: 1/5 (low) No users/revenue disclosed and zero public footprint; Pomodoro/focus tools are notoriously low-retention. No evidence of PMF.
- unit_economics: 1/5 (low) $1.5K price + no metrics = almost certainly pre-revenue; monetizing focus timers is hard (low willingness to pay).
- moat_channel: 1/5 (med) Red-ocean category (Pomofocus, Forest, Session, Flow, Todoist, Notion); no moat, and no visible acquisition channel.
- growth_market: 2/5 (med) Huge but saturated market; an AI 'focus coach' angle is possible but everyone is doing it.
- tech_ip: 3/5 (low) It's a web app the team could own — but the site returned an empty shell and the codebase/stack/ownership is unverified (could be a template).
- deal_quality: 2/5 (low) $1,500 is cheap in absolute terms, but poor value for a team that can build a Pomodoro app in a weekend — unless real users come with it.
- seller_risk: 2/5 (med) 'Offloading distracting projects to focus on e-commerce' = abandoned hobby; withholding all metrics strongly implies ~no revenue.

## Key unknowns
- Are there ANY real, engaged users / any revenue? This is the whole deal — with none, you're buying rebuildable code. Swings from Pass to Promising.
- Retention: do users come back after a few weeks? Pomodoro tools churn hard; decisive for whether there's a business.
- Is there a functioning app at all (fetch returned an empty shell) and is 100% of the code owned + transferable (not a paid template/boilerplate)?
- Any acquisition channel with traction? Zero public footprint suggests none.
- Ongoing costs/liabilities (hosting, APIs) and whether any paying user transfers.

## Seller questions
1. The decisive one: current registered users, MAU/WAU, and any revenue/MRR — screen-shared from analytics + Stripe/billing. (PMF + unit economics + revenue verification)
2. Retention: of users who sign up, what share are still active at 4 weeks? (retention)
3. Where do users come from and how many visits/month — organic, Product Hunt, ads, none? (channel; there's no public footprint, so this reveals whether any distribution exists)
4. Exactly what transfers: repo/codebase, domain, and the user database — and is 100% of the code owned by you (no license-limited template/boilerplate)? What's the stack/hosting? (tech + IP transferability)
5. How long has it been live, is there any monetization today, and why sell at $1,500? (adverse selection)
6. Monthly running costs and any third-party dependencies (auth, payments, APIs)? (economics)
